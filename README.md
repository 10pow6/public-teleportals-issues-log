# TelePortals — Public Issue Tracker

Welcome! This is the **public bug & feedback tracker** for **TelePortals** — the URL shortener with a management portal, marketplace, Portal Coin economy, and tiered analytics, available at [tp.city](https://tp.city) and [tp.land](https://tp.land).

This repository contains **no application source code**. It exists so users can report bugs, request features, and track what we're working on — all in the open.

---

## 🐛 Found a bug? Have an idea?

[**→ Open a new issue**](https://github.com/10pow6/public-teleportals-issues-log/issues/new/choose)

Pick the right template and fill in as much as you can:

| Template | Use it when… |
|----------|--------------|
| 🐛 **Bug Report** | A short link, the dashboard, billing, the marketplace, or the API isn't working right. |
| ✨ **Feature Request** | You have an idea for a new capability or a quality‑of‑life improvement. |

Before opening one, please [**search existing issues**](https://github.com/10pow6/public-teleportals-issues-log/issues?q=is%3Aissue) — if your bug or idea already exists, add a 👍 or a comment instead of opening a duplicate. It helps us prioritize.

## 🔒 Security issues

**Do not** open a public issue for security vulnerabilities. See [**SECURITY.md**](SECURITY.md) for how to disclose privately.

> ⚠️ Reports involving the **Integration API**, Portal Coins, billing, or anything that could expose another user's data or links should be treated as **security‑sensitive** — disclose those privately, not here.

## 💬 Want to chat instead?

Join the community on **Discord** (linked from the TelePortals site) for live discussion, help, and announcements. GitHub issues are best for things we need to track and fix; Discord is best for questions and conversation.

## How we handle issues

- We triage new issues and apply labels (e.g. `bug`, `enhancement`, `needs-info`, `confirmed`, `wont-fix`).
- If we need more detail we'll add `needs-info` and comment — please reply, or we may close the issue after a while.
- Fixed issues are closed with a note. Shipped features are closed when they go live.
- This is the canonical place to see **what's reported and what's planned**.

## Tips for a great report

1. **One issue per report.** Separate bugs into separate issues.
2. **Be specific.** "Redirect 404s on tp.land custom codes" beats "links are broken."
3. **Include steps** to reproduce, what you expected, and what actually happened.
4. **Add screenshots** when the bug is visual — but **redact any private data** (tokens, full URLs you don't want public, billing details).
5. **Tell us your setup** — browser, device, which domain (tp.city / tp.land), and your account tier if relevant.

Thanks for helping make TelePortals better! 🚀
